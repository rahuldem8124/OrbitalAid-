# Plan: Add Draggable Detail Popup to Alerts Page

## Context

The alerts page (`src/app/alerts/page.tsx`) currently shows a simple list of alerts with an Acknowledge button. The user wants each alert to be clickable, opening a draggable detail popup (using the existing `DraggablePanel`) that shows **more detail** about the alert — including its associated conjunction event — and includes a button to redirect to the maneuvers page.

**Key insight**: The `Alert` type has `conjunction_event_id`, so the detail popup can fetch the full conjunction via `fetchConjunction()` and display conjunction data alongside alert-specific info. The popup needs a "View Maneuvers" button that navigates to `/maneuvers`.

**Current structure**: The page is a server component fetching alerts server-side. Following the established pattern, we'll extract the interactive list into a client component (`AlertsListClient.tsx`) and keep the page as a server component.

## Files to Modify

| File | Action |
|------|--------|
| `frontend/src/components/dashboard/AlertDetailPanel.tsx` | **Create** — alert detail popup with conjunction info + redirect button |
| `frontend/src/components/dashboard/AlertsListClient.tsx` | **Create** — client wrapper for clickable alert cards |
| `frontend/src/app/alerts/page.tsx` | **Modify** — use AlertsListClient instead of inline rendering |

## 1. Create `AlertDetailPanel.tsx`

Props: `{ alert: Alert | null; onClose: () => void }`

Takes the full `Alert` object directly (no fetch needed — the page already has all alert fields from server-side fetch). Fetches only the associated conjunction via `fetchConjunction(alert.conjunction_event_id)`.

**Layout inside DraggablePanel** (title: "Alert Detail"):

1. **Alert header** — severity badge (colored by severity using the same color scheme as RiskIndicator), created_at timestamp, acknowledged status (checkmark if acknowledged, "Unacknowledged" label if not)
2. **Alert message** — the full `message` text
3. **Channels sent** — display `channels_sent` if present, or "N/A" if null
4. **Associated Conjunction** — fetch via `fetchConjunction(alert.conjunction_event_id)`, show:
   - Object names (A ↔ B)
   - Miss distance, TCA, relative velocity, Pc
   - Risk tier badge
   - Loading spinner while fetching
5. **Actions row**:
   - **Acknowledge button** (only if not already acknowledged) — reuse `acknowledgeAlert()` logic from AlertActions.tsx, show inline confirmation ("Acknowledged ✓") for 2 seconds, update the local alert state in place (not `router.refresh()` since this is a popup)
   - **"View Maneuvers" button** — uses `next/navigation`'s `useRouter` to navigate to `/maneuvers`, then closes the popup

## 2. Create `AlertsListClient.tsx`

Same pattern as `ConjunctionsTableClient.tsx` and `ManeuversListClient.tsx`:
- `"use client"` directive
- Props: `{ alerts: Alert[] }`
- `useState<string | null>` for `selected` (stores alert ID)
- Renders the grid of `<GlassPanel>` alert cards (same markup as current page)
- Each card gets `onClick={() => setSelected(a.id)}` with `cursor-pointer`
- Keep `<AlertActions>` inside each card for quick acknowledge from the list
- Renders `<AlertDetailPanel alert={selectedAlert} onClose={() => setSelected(null)} />` as a sibling, where `selectedAlert` is found by filtering `alerts` by the selected ID

## 3. Update `alerts/page.tsx`

- Import `AlertsListClient` instead of rendering inline
- Render `<AlertsListClient alerts={alerts} />`
- Page stays as a server component with `force-dynamic`

No new API functions needed — the alert data is already fully available from the server-side `fetchAlerts()` call.
